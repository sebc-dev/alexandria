## Notes

1. **Intentional Design:** `./quality` uses `mvn test` (fast, unit only) while CI uses `mvn verify` (comprehensive). This balances local iteration speed with CI thoroughness.

2. **Code Structure:** Quality script duplicates awk parsing logic rather than calling coverage/mutation scripts. This is intentional for self-contained execution.

3. **Badge Bootstrap:** Badges will appear after first push to main. `.github/badges/.gitkeep` ensures directory is tracked.
