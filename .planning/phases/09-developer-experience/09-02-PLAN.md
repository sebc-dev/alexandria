Create executable shell script `alexandria` at project root.

Requirements:
- Use #!/bin/sh for POSIX portability (not #!/bin/bash)
- Set -e for exit on error
- Resolve script directory for running from anywhere
- Show help message if no arguments provided
- Check Docker is installed (command -v docker)
- Check container is running (docker compose ps --status running)
- Execute via docker compose exec with:
  - -T flag to disable TTY (works in scripts/CI)
  - -e SPRING_PROFILES_ACTIVE=cli,docker to override HTTP profile
  - Pass all arguments with "$@"
- Use exec for proper signal handling

Help message should show:
- Available commands: index, search, status, clear
- Example usage for each command
- Note about container needing to be running

Error messages:
- "Docker is not installed" if docker command missing
- "Alexandria container is not running. Start with: docker compose up -d" if container down

Implementation from research:
```sh
#!/bin/sh
set -e

SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
cd "$SCRIPT_DIR"

# Help if no args
if [ $# -eq 0 ]; then
    cat <<EOF
Alexandria - RAG system for personal documentation

Usage: ./alexandria <command> [options]

Commands:
  index --path <dir>    Index markdown files from directory
  search --query <text> Search indexed documentation
  status                Show database status
  clear --force         Clear all indexed data

Examples:
  ./alexandria index --path /docs
  ./alexandria search --query "how to configure logging"
  ./alexandria status

Note: Container must be running (docker compose up -d)
EOF
    exit 0
fi

# Check Docker
command -v docker >/dev/null 2>&1 || { echo "Error: Docker is not installed" >&2; exit 1; }

# Check container running
if ! docker compose ps --status running 2>/dev/null | grep -q alexandria-app; then
    echo "Error: Alexandria container is not running." >&2
    echo "Start it with: docker compose up -d" >&2
    exit 1
fi

# Execute in container
exec docker compose exec -T -e SPRING_PROFILES_ACTIVE=cli,docker app \
    java org.springframework.boot.loader.launch.JarLauncher "$@"
```

```markdown
## Quick Start with Docker

The easiest way to run Alexandria - no Java or Maven installation required.

### Prerequisites

- Docker 20.10+ with Docker Compose V2
- 4GB available RAM (2GB for app + 2GB for database)
- Git

### 1. Clone and Configure

\`\`\`bash
git clone https://github.com/[owner]/alexandria.git
cd alexandria

# Create your configuration
cp .env.example .env

# Edit .env to set your documentation path
# DOCS_PATH=/path/to/your/docs
\`\`\`

### 2. Start Services

\`\`\`bash
docker compose up -d

# Wait for services to be healthy (~2 minutes for ONNX model loading)
docker compose ps
\`\`\`

### 3. Index Your Documentation

\`\`\`bash
./alexandria index --path /docs
\`\`\`

### 4. Search

\`\`\`bash
./alexandria search --query "how to configure logging"
\`\`\`

### CLI Commands (Docker)

\`\`\`bash
./alexandria index --path /docs      # Index documentation
./alexandria search --query "text"   # Search documentation
./alexandria status                  # Show database status
./alexandria clear --force           # Clear all indexed data
\`\`\`
```
