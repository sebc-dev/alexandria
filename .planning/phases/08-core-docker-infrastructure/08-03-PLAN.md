  <name>Task 1: Add app service to docker-compose.yml</name>
  <files>docker-compose.yml</files>
  <action>
Update docker-compose.yml to add the alexandria-app service:

```yaml
services:
  postgres:
    build: .
    container_name: rag-postgres
    ports:
      - "5432:5432"
    environment:
      POSTGRES_USER: alexandria
      POSTGRES_PASSWORD: alexandria
      POSTGRES_DB: alexandria
    volumes:
      # Data persistence
      - ./data:/var/lib/postgresql/data
      # Liquibase changelogs for init scripts
      - ./src/main/resources/db/changelog:/docker-entrypoint-initdb.d/changelog:ro
      # Init script to run Liquibase changelogs
      - ./scripts/init-db.sh:/docker-entrypoint-initdb.d/01-init-db.sh:ro
    # Shared memory for HNSW index operations
    shm_size: 3g
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U alexandria -d alexandria"]
      interval: 10s
      timeout: 5s
      retries: 5
      start_period: 30s
    restart: unless-stopped

  app:
    build:
      context: .
      dockerfile: Dockerfile.app
    container_name: alexandria-app
    ports:
      - "8080:8080"
    environment:
      # Spring profiles for HTTP/SSE transport
      SPRING_PROFILES_ACTIVE: http,docker
      # Database connection (uses service name as hostname)
      DB_HOST: postgres
      DB_PORT: 5432
      DB_NAME: alexandria
      DB_USER: alexandria
      DB_PASSWORD: alexandria
      # Logging level (optional override)
      LOG_LEVEL: INFO
    volumes:
      # Documentation directory (read-only mount)
      - ${DOCS_PATH:-./docs}:/docs:ro
    depends_on:
      postgres:
        condition: service_healthy
    healthcheck:
      # Use wget instead of curl (smaller image, no curl installed)
      test: ["CMD", "wget", "-q", "--spider", "http://localhost:8080/actuator/health"]
      interval: 30s
      timeout: 10s
      retries: 3
      # Extended start_period for ONNX model loading (can take 60-120s)
      start_period: 120s
    restart: unless-stopped
    # Memory limit (ONNX model needs ~500MB, app needs ~500MB, buffer for GC)
    mem_limit: 2g
```

Key configuration decisions:
1. **Service name**: `app` with container name `alexandria-app`
2. **Ports**: 8080 exposed for MCP HTTP/SSE endpoint
3. **depends_on with condition**: Waits for postgres healthcheck to pass
4. **Health check**: Uses wget (not curl) since JRE image doesn't have curl
5. **start_period: 120s**: Accommodates ONNX model download/load time
6. **Volume mount**: `${DOCS_PATH:-./docs}` allows override via .env (Phase 9)
7. **mem_limit**: 2GB default prevents OOM on small hosts
8. **restart: unless-stopped**: Survives host reboot but stays down on OOM


2. Start services:
```bash

```

3. Monitor startup:
```bash
docker compose logs -f app
```
Wait for "Started AlexandriaApplication" or equivalent startup message.

4. Verify health:
```bash
docker compose ps
```
Both services should show "healthy" status.

5. Test MCP endpoint:
```bash
curl -v http://localhost:8080/actuator/health
```
Should return 200 with {"status":"UP"}

6. Test graceful shutdown:
```bash
time docker compose down
```
Should complete in < 35 seconds (graceful shutdown + 5s buffer, not 10s SIGKILL timeout).

Note: First startup may take 2-3 minutes due to:
- Docker image build (if not cached)
- ONNX model download (first run only)
- Database initialization
