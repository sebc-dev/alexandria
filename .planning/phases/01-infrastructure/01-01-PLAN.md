  <action>
Create a Dockerfile based on `pgvector/pgvector:0.8.1-pg17` that compiles Apache AGE from source.

Key requirements:
- Install build dependencies (git, build-essential, postgresql-server-dev-17)
- Clone AGE from `https://github.com/apache/age.git` tag `PG17/v1.6.0-rc0`
- Build and install AGE extension
- Create symlink for non-superuser loading: `ln -s age.so /usr/lib/postgresql/17/lib/plugins/age.so`
- Copy custom postgresql.conf

Create `postgresql.conf` with:
```
shared_preload_libraries = 'age'
maintenance_work_mem = 512MB
```

The Dockerfile should use multi-stage build to keep final image small (compile in builder stage, copy only artifacts to final stage).
  </action>

`docker build -t rag-postgres .` succeeds without errors
  </verify>
  <done>
Dockerfile builds successfully, produces image with pgvector pre-installed and AGE compiled
  </done>

`db.changelog-master.yaml`:
```yaml

  - include:
      file: changes/001-extensions.sql
      relativeToChangelogFile: true
  - include:
      file: changes/002-schema.sql
      relativeToChangelogFile: true
```

`001-extensions.sql` (with Liquibase header):
- CREATE EXTENSION IF NOT EXISTS vector
- CREATE EXTENSION IF NOT EXISTS age
- Load AGE: `LOAD 'age';`
- Set search path: `SET search_path = ag_catalog, "$user", public;`
- Create graph: `SELECT create_graph('alexandria');`

`002-schema.sql` (with Liquibase header):
- Table `documents`:
  - id UUID PRIMARY KEY DEFAULT gen_random_uuid()
  - path TEXT NOT NULL UNIQUE
  - title TEXT
  - category TEXT
  - tags TEXT[] DEFAULT '{}'
  - content_hash TEXT NOT NULL (SHA256)
  - frontmatter JSONB DEFAULT '{}'
  - created_at TIMESTAMPTZ DEFAULT now()
  - updated_at TIMESTAMPTZ DEFAULT now()

- Table `chunks`:
  - id UUID PRIMARY KEY DEFAULT gen_random_uuid()
  - document_id UUID REFERENCES documents(id) ON DELETE CASCADE
  - parent_chunk_id UUID REFERENCES chunks(id) ON DELETE CASCADE (nullable, for child chunks)
  - chunk_type TEXT NOT NULL CHECK (chunk_type IN ('parent', 'child'))
  - content TEXT NOT NULL
  - embedding vector(384)
  - position INTEGER NOT NULL
  - created_at TIMESTAMPTZ DEFAULT now()

- Index HNSW on chunks.embedding: `CREATE INDEX ON chunks USING hnsw (embedding vector_cosine_ops) WITH (m = 16, ef_construction = 100);`
- Index on chunks.document_id
- Index GIN on documents.tags
- Full-text index: `CREATE INDEX ON chunks USING gin(to_tsvector('simple', content));`

Use snake_case for all identifiers as per CONTEXT.md decision.
  </action>

  </verify>
