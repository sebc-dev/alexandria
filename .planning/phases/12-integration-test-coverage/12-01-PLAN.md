<task type="auto">
  <name>Task 1: Add JaCoCo Integration Test Goals</name>
  <files>pom.xml</files>
  <action>
Add four new executions to the existing jacoco-maven-plugin configuration:

1. **prepare-agent-integration** (before integration-test phase):
```xml
<execution>
    <id>prepare-agent-integration</id>
    <goals>
        <goal>prepare-agent-integration</goal>
    </goals>
</execution>
```
Default phase: pre-integration-test (automatic)
Default destFile: ${project.build.directory}/jacoco-it.exec (automatic)

2. **report-integration** (generate IT-only report):
```xml
<execution>
    <id>report-integration</id>
    <phase>post-integration-test</phase>
    <goals>
        <goal>report-integration</goal>
    </goals>
</execution>
```
Default outputDirectory: ${project.reporting.outputDirectory}/jacoco-it (automatic)

3. **merge-results** (combine unit + IT data):
```xml
<execution>
    <id>merge-results</id>
    <phase>post-integration-test</phase>
    <goals>
        <goal>merge</goal>
    </goals>
    <configuration>
        <fileSets>
            <fileSet>
                <directory>${project.build.directory}</directory>
                <includes>
                    <include>jacoco.exec</include>
                    <include>jacoco-it.exec</include>
                </includes>
            </fileSet>
        </fileSets>
        <destFile>${project.build.directory}/jacoco-merged.exec</destFile>
    </configuration>
</execution>
```
CRITICAL: Must bind to post-integration-test (default is generate-resources which runs before tests)

4. **report-merged** (generate combined report):
```xml
<execution>
    <id>report-merged</id>
    <phase>verify</phase>
    <goals>
        <goal>report</goal>
    </goals>
    <configuration>
        <dataFile>${project.build.directory}/jacoco-merged.exec</dataFile>
        <outputDirectory>${project.reporting.outputDirectory}/jacoco-merged</outputDirectory>
    </configuration>
</execution>
```

Place these after the existing "report" execution in the jacoco-maven-plugin.
  </action>
  <verify>Run `mvn help:effective-pom | grep -A5 "prepare-agent-integration"` to confirm execution exists</verify>
  <done>pom.xml has all four new JaCoCo executions</done>
</task>

<task type="auto">
  <name>Task 2: Add argLine to Failsafe Plugin</name>
  <files>pom.xml</files>
  <action>
Add the argLine configuration to maven-failsafe-plugin to enable JaCoCo instrumentation during integration tests.

Current configuration:
```xml
<plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-failsafe-plugin</artifactId>
    <configuration>
        <includes>
            <include>**/*IT.java</include>
        </includes>
    </configuration>
    ...
</plugin>
```

Update to:
```xml
<plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-failsafe-plugin</artifactId>
    <configuration>
        <includes>
            <include>**/*IT.java</include>
        </includes>
        <!-- @{argLine} = JaCoCo agent, then Mockito agent for Java 21+ -->
        <argLine>@{argLine} -XX:+EnableDynamicAgentLoading</argLine>
    </configuration>
    ...
</plugin>
```

CRITICAL: Without this, jacoco-it.exec will be empty/missing (Pitfall 1 from research).
The @{argLine} syntax uses Maven late property evaluation to pick up the JaCoCo agent string.
  </action>
  <verify>Run `mvn help:effective-pom | grep -B2 -A4 "maven-failsafe-plugin"` and confirm argLine is present</verify>
  <done>Failsafe plugin has argLine with @{argLine} and -XX:+EnableDynamicAgentLoading</done>
</task>
