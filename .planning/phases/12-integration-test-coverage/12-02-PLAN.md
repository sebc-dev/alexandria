<task type="auto">
  <name>Task 1: Enable Integration Tests in CI</name>
  <files>.github/workflows/ci.yml</files>
  <action>
Update the build job to run full `mvn verify` instead of skipping integration tests.

Current command:
```yaml
- name: Build and run unit tests
  run: mvn -B -T 1C verify -DskipITs
```

Change to:
```yaml
- name: Build and run all tests
  run: mvn -B verify
```

Rationale:
- Remove `-DskipITs` to enable integration tests
- Remove `-T 1C` (thread per core) because Testcontainers startup has race conditions with parallel builds
- Keep `-B` for batch mode (no interactive input)

Note: Ubuntu-latest has Docker pre-installed, so Testcontainers works out of the box.
  </action>
  <verify>Grep ci.yml for "mvn -B verify" without -DskipITs</verify>
  <done>ci.yml runs full mvn verify including integration tests</done>
</task>

<task type="auto">
  <name>Task 2: Add JaCoCo Report Artifact Upload</name>
  <files>.github/workflows/ci.yml</files>
  <action>
Add a step to upload JaCoCo coverage reports after the build step.

Add after the existing "Upload test results" step:
```yaml
- name: Upload JaCoCo coverage reports
  uses: actions/upload-artifact@v4
  if: always()
  with:
    name: jacoco-reports
    path: |
      target/site/jacoco/
      target/site/jacoco-it/
      target/site/jacoco-merged/
    retention-days: 7
```

Key points:
- `if: always()` - Upload even if tests fail (for debugging)
- Use v4 of upload-artifact (latest stable)
- Multi-path syntax with pipe to include all three report directories
- 7-day retention is sufficient for CI debugging
- Name "jacoco-reports" for clear identification

Also update the existing test results artifact to include failsafe reports:
```yaml
- name: Upload test results
  uses: actions/upload-artifact@v4
  if: always()
  with:
    name: test-results
    path: |
      target/surefire-reports/
      target/failsafe-reports/
    retention-days: 7
```
  </action>
  <verify>Grep ci.yml for "jacoco-reports" and "upload-artifact"</verify>
  <done>ci.yml uploads JaCoCo reports (jacoco/, jacoco-it/, jacoco-merged/) as artifacts</done>
</task>
