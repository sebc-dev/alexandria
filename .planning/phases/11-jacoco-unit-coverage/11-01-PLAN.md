<task type="auto">
  <name>Task 1: Configure JaCoCo Maven Plugin</name>
  <files>pom.xml</files>
  <action>
Add JaCoCo Maven plugin to pom.xml with proper argLine merging:

1. Add empty `<argLine></argLine>` property in `<properties>` section (prevents "Could not find or load main class @{argLine}" error when running tests without coverage)

2. Add jacoco-maven-plugin (version 0.8.14) to build/plugins with two executions:
   - `prepare-agent` goal (default phase: initialize)
   - `report` goal bound to `test` phase (NOT default `verify`) so `mvn test` generates reports

3. Modify existing maven-surefire-plugin argLine from:
   `<argLine>-XX:+EnableDynamicAgentLoading</argLine>`
   to:
   `<argLine>@{argLine} -XX:+EnableDynamicAgentLoading</argLine>`

   The `@{argLine}` syntax enables late property evaluation, allowing JaCoCo to prepend its agent.

Note: Keep the comment about Mockito agent for Java 21+.

Reference configuration from 11-RESEARCH.md "Complete pom.xml Configuration" section.
  </action>
  <verify>
Run: `mvn clean test`

Expected:
- Tests pass (Mockito still works with argLine merging)
- `target/jacoco.exec` exists (binary coverage data)
- `target/site/jacoco/index.html` exists (HTML report)
- `target/site/jacoco/jacoco.xml` exists (XML report)
  </verify>
  <done>
- JaCoCo plugin added with prepare-agent and report goals
- Report goal bound to test phase
- Surefire argLine uses @{argLine} pattern
- All unit tests pass
- HTML and XML reports generated by mvn test
  </done>
</task>

<task type="auto">
  <name>Task 2: Create Coverage Convenience Script</name>
  <files>coverage</files>
  <action>
Create executable bash script `./coverage` in project root that:

1. Runs `mvn test -q` (quiet mode for cleaner output)
2. Checks if `target/site/jacoco/jacoco.xml` exists, exits with error if not
3. Parses XML report using xmllint to extract coverage counters
4. Displays formatted coverage summary for LINE, BRANCH, METHOD, CLASS types
5. Shows path to full HTML report

Script requirements:
- Shebang: `#!/usr/bin/env bash`
- Use `set -euo pipefail` for safety
- Calculate percentages: covered * 100 / (missed + covered)
- Format output aligned with printf
- Handle case where total is 0 (show "N/A")

Reference the coverage script example from 11-RESEARCH.md "Coverage Script - Parsing jacoco.xml with xmllint" section.

Make script executable: chmod +x coverage
  </action>
  <verify>
Run: `./coverage`

Expected output format:
```
Coverage Summary
================
LINE        :  XX% (covered/total)
BRANCH      :  XX% (covered/total)
METHOD      :  XX% (covered/total)
CLASS       :  XX% (covered/total)

Full report: target/site/jacoco/index.html
```
  </verify>
  <done>
- coverage script exists and is executable
- Script runs mvn test and displays coverage summary
- Output shows LINE, BRANCH, METHOD, CLASS percentages
- Script exits cleanly on success
  </done>
</task>
