**RetrievalMetrics.java:**
- `final class` with private constructor (utility class pattern)
- No external dependencies - pure Java math

**precisionAtK(List&lt;UUID&gt; retrieved, Set&lt;UUID&gt; relevant, int k):**
- Formula: `|relevant in top K| / K`
- Guard: if k <= 0, return 0.0
- Limit retrieved to first k elements (or all if fewer than k)
- Count matches with relevant set
- Return count / k (NOT count / actual retrieved size)

**recallAtK(List&lt;UUID&gt; retrieved, Set&lt;UUID&gt; relevant, int k):**
- Formula: `|relevant in top K| / |all relevant|`
- Guard: if relevant.isEmpty(), return 0.0
- Limit retrieved to first k elements
- Count matches with relevant set
- Return count / relevant.size()

**reciprocalRank(List&lt;UUID&gt; retrieved, Set&lt;UUID&gt; relevant):**
- Formula: `1 / rank of first relevant result`
- Iterate through retrieved (0-indexed)
- On first match with relevant: return 1.0 / (index + 1)
- If no match found: return 0.0
- Note: rank is 1-indexed (position 0 = rank 1)

**ndcgAtK(List&lt;UUID&gt; retrieved, Set&lt;UUID&gt; relevant, int k):**
- Formula: `DCG@K / IDCG@K`
- Binary relevance: rel=1 if in relevant set, 0 otherwise
- DCG formula: sum of `rel_i / log2(i + 2)` for i from 0 to k-1
  - Use `Math.log(i + 2) / Math.log(2)` for log base 2
  - Position 0 uses log2(2) = 1, position 1 uses log2(3), etc.
- IDCG: perfect ranking with all relevant first
  - idealLimit = min(k, relevant.size())
  - Same formula but all positions have rel=1
- Guard: if no relevant docs in top k, return 0.0
- Return dcg / idcg

**IMPORTANT edge cases to handle:**
- Empty retrieved list: return 0.0 for all metrics
- Empty relevant set: return 0.0 for recall, mrr, ndcg (precision is k/k=0 if no matches)
- k larger than retrieved size: use actual retrieved size
- Division by zero: always check before dividing
