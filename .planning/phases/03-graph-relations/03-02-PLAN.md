    3. Inner class LinkVisitor extends AbstractVisitor:
       - Override visit(Link link) method
       - Filter: only process links where destination:
         - Is not null or empty
         - Does not start with http://, https://, mailto:
         - Does not start with # (anchor)
         - Ends with .md
       - Extract link text from child Text nodes
       - Collect into List<ExtractedLink>
    4. Public method: List<ExtractedLink> extractLinks(String markdownContent)
       - Parse with CommonMark Parser
       - Apply LinkVisitor
       - Return immutable list
    5. Public method: Optional<Path> resolveLink(Path sourceFile, String relativePath)
       - Get parent directory of source file
       - Resolve relative path against parent
       - Normalize to handle ../
       - Return Optional.of(resolved) or Optional.empty() on error

    Use @Component annotation for Spring integration.

    Create CrossReferenceExtractorTest:
    - Test extractLinks with various markdown content:
      - Single link: [text](other.md)
      - Multiple links in same content
      - External links (https://...) are ignored
      - Anchor links (#section) are ignored
      - Links in code blocks should still be detected (CommonMark parses them)
      - Links without .md extension are ignored
      - Empty link text still captures destination
    - Test resolveLink:
      - Same directory: other.md
      - Subdirectory: sub/other.md
      - Parent directory: ../other.md
      - Complex path: ../sibling/doc.md
