<task type="auto">
  <name>Task 1: Add VictoriaMetrics service with eval profile</name>
  <files>
    - docker-compose.yml
    - monitoring/victoriametrics/scrape.yaml
  </files>
  <action>
1. Create `monitoring/victoriametrics/scrape.yaml` with scrape config:
   ```yaml
   scrape_configs:
     - job_name: 'alexandria'
       scrape_interval: 15s
       static_configs:
         - targets:
           - app:8080
       metrics_path: /actuator/prometheus

     - job_name: 'victoriametrics'
       scrape_interval: 15s
       static_configs:
         - targets:
           - localhost:8428
   ```

2. Add VictoriaMetrics service to docker-compose.yml:
   ```yaml
   victoriametrics:
     image: victoriametrics/victoria-metrics:v1.114.0
     container_name: alexandria-vm
     profiles: ["eval"]
     networks:
       - internal
     volumes:
       - vm-data:/victoria-metrics-data
       - ./monitoring/victoriametrics/scrape.yaml:/etc/vm/scrape.yaml:ro
     command:
       - "-storageDataPath=/victoria-metrics-data"
       - "-retentionPeriod=30d"
       - "-promscrape.config=/etc/vm/scrape.yaml"
     healthcheck:
       test: ["CMD", "wget", "-qO-", "http://localhost:8428/health"]
       interval: 10s
       timeout: 5s
       retries: 5
       start_period: 10s
     restart: unless-stopped
   ```

3. Add `vm-data` to volumes section at bottom of docker-compose.yml:
   ```yaml
   volumes:
     vm-data:
   ```

Note: Use wget for health check (VictoriaMetrics image includes it, not curl).
  </action>
  <verify>
    docker compose --profile eval config | grep -A 30 "victoriametrics:"
    # Should show service with profiles: ["eval"], health check, and volume mount
  </verify>
  <done>VictoriaMetrics service defined with eval profile, scrape config mounted, health check configured</done>
</task>

<task type="auto">
  <name>Task 2: Add Grafana service with datasource provisioning</name>
  <files>
    - docker-compose.yml
    - monitoring/grafana/provisioning/datasources/datasources.yaml
  </files>
  <action>
1. Create directory structure:
   ```bash
   mkdir -p monitoring/grafana/provisioning/datasources
   mkdir -p monitoring/grafana/provisioning/dashboards
   mkdir -p monitoring/grafana/provisioning/alerting
   ```

2. Create `monitoring/grafana/provisioning/datasources/datasources.yaml`:
   ```yaml
   apiVersion: 1
   datasources:
     - name: VictoriaMetrics
       type: prometheus
       access: proxy
       url: http://victoriametrics:8428
       isDefault: true
       editable: false
   ```

3. Add Grafana service to docker-compose.yml:
   ```yaml
   grafana:
     image: grafana/grafana:11.6.0
     container_name: alexandria-grafana
     profiles: ["eval"]
     networks:
       - internal
     ports:
       - "3000:3000"
     environment:
       - GF_SECURITY_ADMIN_USER=admin
       - GF_SECURITY_ADMIN_PASSWORD=admin
       - GF_AUTH_ANONYMOUS_ENABLED=true
       - GF_AUTH_ANONYMOUS_ORG_ROLE=Viewer
     volumes:
       - grafana-data:/var/lib/grafana
       - ./monitoring/grafana/provisioning:/etc/grafana/provisioning:ro
     depends_on:
       victoriametrics:
         condition: service_healthy
     healthcheck:
       test: ["CMD", "wget", "-q", "--spider", "http://localhost:3000/api/health"]
       interval: 10s
       timeout: 5s
       retries: 5
       start_period: 30s
     restart: unless-stopped
   ```

4. Add `grafana-data` to volumes section.

Note: Grafana depends on VictoriaMetrics with service_healthy condition for proper startup ordering.
  </action>
  <verify>
    docker compose --profile eval config | grep -A 40 "grafana:"
    # Should show Grafana service depending on VictoriaMetrics with condition: service_healthy
  </verify>
  <done>Grafana service defined with VictoriaMetrics datasource provisioned, depends on VictoriaMetrics health</done>
