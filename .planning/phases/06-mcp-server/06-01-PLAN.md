<task type="auto">
  <name>Task 2: Create MCP profile configuration</name>
  <files>src/main/resources/application-mcp.yml</files>
  <action>
Create `application-mcp.yml` for STDIO transport configuration. This profile is activated when running as MCP server for Claude Code.

Content:
```yaml
# MCP Server profile - STDIO transport for Claude Code integration
spring:
  main:
    # Disable web server for STDIO transport
    web-application-type: none
    # Disable banner to prevent stdout pollution
    banner-mode: off

  ai:
    mcp:
      server:
        name: alexandria
        version: 0.1.0
        # SYNC type for blocking database operations
        type: SYNC
        # STDIO transport for Claude Code
        stdio: true

# File logging only - console output breaks STDIO protocol
logging:
  pattern:
    console: ""
  file:
    name: logs/alexandria-mcp.log
  level:
    fr.kalifazzia.alexandria: INFO
    org.springframework.ai.mcp: DEBUG
```

Key points:
- `web-application-type: none` prevents Tomcat from starting
- `banner-mode: off` prevents Spring banner on stdout
- `logging.pattern.console: ""` silences console logging (breaks STDIO)
- `type: SYNC` matches existing synchronous services (SearchService, IngestionService)
- File logging preserves debug capability without corrupting STDIO
  </action>
  <verify>File exists at src/main/resources/application-mcp.yml with all required properties</verify>
  <done>MCP profile configuration exists with STDIO transport settings</done>
</task>

```java
 * Returns all distinct categories from indexed documents.

 *
 * @return List of unique category names (excluding null values)
 */
List<String> findDistinctCategories();
```

2. Add to JdbcDocumentRepository implementation:
```java
@Override
public Optional<Document> findById(UUID id) {
    String sql = """
            SELECT id, path, title, category, tags, content_hash, frontmatter, created_at, updated_at
            FROM documents
            WHERE id = ?
            """;

    List<Document> results = jdbcTemplate.query(sql, documentRowMapper, id);
    return results.isEmpty() ? Optional.empty() : Optional.of(results.getFirst());
}

@Override
public List<String> findDistinctCategories() {
    String sql = """
            SELECT DISTINCT category
            FROM documents
            WHERE category IS NOT NULL
            ORDER BY category
            """;

    return jdbcTemplate.queryForList(sql, String.class);
}
```

Pattern notes:
- `findById` follows the same pattern as existing `findByPath`
- `findDistinctCategories` returns sorted list, excludes nulls
