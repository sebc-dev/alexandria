### Flow Details

**Flow 1: Index Documentation**
```
User (CLI/MCP) -> AlexandriaCommands/Tools -> IngestionService
  -> MarkdownParser -> HierarchicalChunker -> EmbeddingGenerator
  -> DocumentRepository + ChunkRepository (PostgreSQL)
  -> DocumentIngestedEvent -> GraphOperationsEventListener (AGE)
```

**Flow 2: Search via MCP**
```
Claude Code -> search_docs() -> AlexandriaTools
  -> SearchService.hybridSearch()
  -> EmbeddingGenerator.embed(query)
  -> JdbcSearchRepository.hybridSearch() (pgvector + tsvector RRF)
  -> GraphRepository.findRelatedDocuments() (AGE Cypher)
  -> SearchResultDto[] -> JSON response
```

**Flow 3: Search via CLI**
```
Terminal -> search --query -> AlexandriaCommands
  -> SearchService.hybridSearch()
  -> [same as Flow 2]
  -> Formatted text output
```

**Flow 4: Get Document by ID**
```
Claude Code -> get_doc(uuid) -> AlexandriaTools
  -> DocumentRepository.findById()
  -> DocumentDto -> JSON response
```

**Flow 5: Clear and Reindex**
```
Terminal -> clear --force -> AlexandriaCommands
  -> GraphRepository.clearAll() (DETACH DELETE)
  -> ChunkRepository.deleteAll()
  -> DocumentRepository.deleteAll()
Terminal -> index --path -> [Flow 1]
```

## Architecture Verification

Hexagonal architecture enforced by ArchUnit:

```
api (MCP, CLI) -> core (services, ports) <- infra (repositories)
```

- API layer: AlexandriaTools, AlexandriaCommands
- Core layer: Services, domain models, port interfaces
- Infra layer: JDBC repositories, LangChain4j integrations

All port interfaces have implementations:
- EmbeddingGenerator -> LangChain4jEmbeddingGenerator
- MarkdownParserPort -> MarkdownParser
- ChunkerPort -> HierarchicalChunker
- DocumentRepository -> JdbcDocumentRepository
- ChunkRepository -> JdbcChunkRepository
- GraphRepository -> AgeGraphRepository
- SearchRepository -> JdbcSearchRepository
