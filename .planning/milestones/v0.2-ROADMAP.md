# Milestone v0.2: Full Docker

**Status:** SHIPPED 2026-01-22
**Phases:** 8-10
**Total Plans:** 6

## Overview

Rendre l'application installable et utilisable via docker compose sans installer de dependances.

## Phases

### Phase 8: Core Docker Infrastructure

**Goal**: Application runs in Docker with HTTP/SSE transport for MCP communication
**Depends on**: Phase 7 (v0.1 complete)
**Plans**: 3 plans

Plans:
- [x] 08-01: HTTP/SSE transport and Spring profile configuration
- [x] 08-02: Multi-stage Dockerfile with layered JAR extraction
- [x] 08-03: Docker Compose service definition with health checks

**Details:**
- Added spring-ai-starter-mcp-server-webmvc for HTTP/SSE transport
- Created application-http.yml and application-docker.yml profiles
- Multi-stage Dockerfile with eclipse-temurin:21-jre runtime
- Layered JAR extraction for optimal Docker layer caching
- Non-root user (uid 10000) for security
- docker-compose.yml with health checks (120s start period for ONNX)
- Service dependency with service_healthy condition

### Phase 9: Developer Experience

**Goal**: Users can easily configure and interact with containerized Alexandria
**Depends on**: Phase 8
**Plans**: 2 plans

Plans:
- [x] 09-01: .dockerignore and .env configuration
- [x] 09-02: CLI wrapper script and README documentation

**Details:**
- .dockerignore reduces build context for faster builds
- .env.example template with all configuration options documented
- MEM_LIMIT configurable via environment variable
- POSIX shell wrapper script `./alexandria` for CLI access
- Docker Quick Start section in README

### Phase 10: CI/CD Pipeline

**Goal**: Docker images automatically published to GitHub Container Registry on release
**Depends on**: Phase 9
**Plans**: 1 plan

Plans:

**Details:**

## Milestone Summary

**Key Decisions:**
- Keep both STDIO and HTTP/SSE MCP starters for profile-based selection
- Use wget instead of curl for health check (JRE image has no curl)
- 120s health check start_period for ONNX model loading
- POSIX sh instead of bash for CLI wrapper portability
- Official Docker GitHub Actions v3/v5/v6 for GHCR publishing

**Issues Resolved:**
- Maven wrapper not present - install Maven directly in Dockerfile
- JarLauncher class not found - add --launcher flag to extract command
- Logback cannot create logs directory - create /application/logs in Dockerfile

**Issues Deferred:**
None

**Technical Debt Incurred:**
None from v0.2

**Inherited from v0.1:**
- MCP tool unit tests missing (wiring verified via service layer tests)
- Integration tests for hybrid search not present (mocked in unit tests)
