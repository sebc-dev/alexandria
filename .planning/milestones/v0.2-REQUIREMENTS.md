# Requirements: Alexandria v0.2 Full Docker

**Defined:** 2026-01-22
**Core Value:** Claude Code peut acceder a ma documentation technique personnelle pendant l'implementation pour respecter mes conventions et bonnes pratiques.

## v0.2 Requirements

Requirements for Docker packaging milestone. Each maps to roadmap phases.

### Docker Infrastructure

- [x] **DOCK-01**: Multi-stage Dockerfile avec JDK build et JRE runtime
- [x] **DOCK-02**: Spring Boot layered JAR extraction pour cache efficiency
- [x] **DOCK-03**: docker-compose.yml avec service app + postgres
- [x] **DOCK-04**: Health check avec depends_on service_healthy condition
- [x] **DOCK-05**: Container execute en tant que non-root user
- [x] **DOCK-06**: Graceful shutdown (exec form ENTRYPOINT + Spring lifecycle)
- [x] **DOCK-07**: Extended start_period (120s) pour accommoder ONNX loading

### MCP Transport

- [x] **MCP-01**: HTTP/SSE transport via spring-ai-starter-mcp-server-webmvc
- [x] **MCP-02**: Port 8080 expose pour clients MCP externes
- [x] **MCP-03**: STDIO transport conserve et configurable via Spring profile

### Configuration

- [x] **CONF-01**: Variables d'environnement pour connexion database (DB_HOST, etc.)
- [x] **CONF-02**: Volume mount pour repertoire documents configurable
- [x] **CONF-03**: Support fichier .env pour configuration overrides
- [x] **CONF-04**: Fichier .env.example avec documentation des options

### Developer Experience

- [x] **DEVX-01**: Script wrapper CLI (`alexandria` command) pour docker exec
- [x] **DEVX-02**: .dockerignore pour optimisation build (exclure target/, .git/, etc.)
- [x] **DEVX-03**: README mis a jour avec instructions installation et usage Docker

### CI/CD

- [x] **CICD-01**: GitHub Actions workflow pour build image Docker
- [x] **CICD-02**: Publication automatique sur GitHub Container Registry (ghcr.io)
- [x] **CICD-03**: Tagging semantique depuis git tags (v0.2.0 -> ghcr.io/user/alexandria:0.2.0)
